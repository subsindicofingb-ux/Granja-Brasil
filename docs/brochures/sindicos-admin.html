Estrutura de interligação multi-condomínio

A Granja Brasil opera como condomínio-mãe (hub administrativo). Condomínios filhos — Jacarandás, Jequitibás, Manacás, Mangabeiras, Cambucás, Jabuticabeiras, Bouganville, Acácias, Cerejeiras e demais — funcionam como blocos residenciais com torres, unidades e moradores próprios, compartilhando serviços centrais.

Hierarquia de dados

  • Condomínio → torres/blocos → unidades → moradores
  • Memberships vinculam perfis a papéis por condomínio
  • Solicitações de cadastro → aprovação → morador + acesso ControlID
  • Blocos de portaria agrupam condomínios com entrada única operacional

Comunicação eficiente Granja ↔ condomínios

  • Fale com o condomínio — canal bidirecional síndico ↔ Granja
  • Avisos direcionados — Granja publica para condomínios específicos
  • Notificações formais — registro oficial à unidade com comprovante
  • E-mail transacional — cadastros, avisos e alertas automáticos

Fluxos operacionais principais

Cadastro de morador

Signup público → fila de solicitações → síndico aprova → morador + membership + ControlID → painel liberado.

Reserva de área comum

Morador solicita → comprovante (se exigido) → aprovação síndico/Granja → uso → devolução com assinatura digital.

Visitante / prestador

Morador autoriza → síndico/portaria valida → portaria consulta na entrada → janela de acesso configurável.

Veículo

Cadastro com placa/TAG/foto → aprovação síndico → validação Granja (global) → liberação ControlID na garagem.

ControlID

Locais de acesso cadastrados → sync automático ao aprovar morador/veículo → worker em background com retry.

Hidrômetros

Portaria registra leituras diárias → alertas de consumo → síndico consulta histórico e médias por unidade.

Segurança e acesso{{APP_URL}}

Controle de acesso e integridade

Gate de aprovação

  • Morador só acessa o painel após aprovação do síndico
  • Vínculo ativo exige registro em moradores (residents)
  • Memberships órfãs removidas automaticamente
  • Tela de autorização pendente até liberação

ControlID — sincronização

  • Locais: portaria, garagem, brinquedoteca, etc.
  • Foto comprimida automaticamente para limite do equipamento
  • Sync ao aprovar, atualizar ou excluir morador/veículo
  • Fila de jobs com retry manual na interface

RLS e multi-tenant

  • Row Level Security por condomínio no Supabase
  • Portaria de bloco enxerga condomínios do mesmo grupo
  • Papéis validados em middleware e server actions

Contato único por unidade

  • E-mail e telefone únicos por unidade (permitido em unidades diferentes)
  • Bloqueio de duplicata na mesma unidade no signup e cadastro
