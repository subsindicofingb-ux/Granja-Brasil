Ecossistema Granja Brasil — visão técnica

A Granja opera como condomínio-mãe (hub). Condomínios filhos — Jacarandás, Jequitibás, Manacás, Mangabeiras, Cambucás, Jabuticabeiras, Bouganville, Acácias, Cerejeiras e demais — mantêm estrutura própria (torres → unidades → moradores) e compartilham serviços centrais, áreas comuns e portaria por bloco.

Hierarquia de dados

  • Condomínio → torres/blocos → unidades → moradores (residents)
  • Memberships vinculam perfil + papel por condomínio
  • Solicitações → aprovação síndico → morador + ControlID
  • Blocos portaria agrupam condomínios com entrada única
  • Reservas e avisos respeitam escopo de torre e condomínio

Isolamento e escopo (RLS)

  • Cada condomínio filho isolado por Row Level Security
  • Admin Granja enxerga todos os filhos via is_granja_staff()
  • Síndico local vê apenas seu condomínio
  • Portaria de bloco vê condomínios do mesmo grupo operacional
  • Super Admin configura hierarquia Ver / Cadastrar / Excluir por papel

Serviços do hub central

Áreas compartilhadas

Salão, quadra e espaços reserváveis entre condomínios do ecossistema, com agenda anti-conflito e comprovante digital.

Dashboard consolidado

Admin Granja monitora pendências, cadastros, reservas e avisos de todos os condomínios filhos em um único painel.

ControlID integrado

Sync automático de moradores e veículos nos locais de acesso — portaria, garagem e brinquedoteca — com fila de retry.

Passo 5 — Camadas de segurança do cadastro à liberação física no ControlID.

Stack tecnológico

Next.js 15 Supabase PostgreSQL Row Level Security ControlID API Resend Vercel Edge

Gate de aprovação

  • Morador só acessa após aprovação do síndico
  • Vínculo exige registro em moradores (residents)
  • Memberships órfãs removidas automaticamente
  • Tela de autorização pendente até liberação

ControlID — sincronização

  • Locais: portaria, garagem, brinquedoteca
  • Foto comprimida para limite do equipamento
  • Sync ao aprovar/atualizar/excluir morador ou veículo
  • Fila de jobs com retry manual na interface

RLS multi-tenant

  • Políticas por condomínio no PostgreSQL
  • Helpers: is_condo_staff, is_granja_staff, is_super_admin
  • Validação dupla: banco + server actions
  • Isolamento garantido mesmo com múltiplos papéis

Cadastro pela portaria

  • Porteiro cadastra morador → conclusão imediata
  • Síndico recebe e-mail de aviso automático
  • Sync ControlID disparado na hora
  • Sem fila de aprovação do Super Admin
